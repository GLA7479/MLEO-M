# BASE — שאילתות דיבאג (`sql/debug/base/`)

קבצי SQL לבדיקות ותחזוקה ב-Supabase / Postgres. **כל קובץ = שאילתה עיקרית אחת.**

## לפני הרצה

- בקבצים עם **`PUT-DEVICE-ID-HERE`** — החלף ב-`device_id` האמיתי (UUID).
- הרץ ב-**SQL Editor** בסביבה המתאימה (מומלץ staging לפני production).

## בטוחים לבדיקות שגרתיות (קריאה / דיווח בלבד)

| קבצים | תיאור |
|--------|--------|
| **01–07** | סקירות, top 20, פעילות 24ש, תקרה יומית, “תקועים”, flatten משאבים |
| **09–12** | audit למשתמש / ספירות / ship / expedition |
| **13, 15, 16** | סוג עמודה, הגדרת reconcile, economy config |
| **17, 21** | צבירה בלי ship, רשימת לא פעילים 30 יום (לפני מחיקה) |

## משנים נתונים — השתמש בזהירות

| קובץ | הערה |
|------|------|
| **08** | `base_reconcile_state` — מריץ לוגיקת שרת ומעדכן מצב |
| **14** | `UPDATE` ל-banked לבדיקה עשרונית |

## הרסניים — **לא** להריץ בלי גיבוי והבנה

| קובץ | מה קורה |
|------|---------|
| **18** | מוחק רק `base_device_state` למכשיר |
| **19** | מוחק audit + state למכשיר |
| **20** | איפוס רך (ברירת מחדל) — **שומר** audit |
| **22** | מוחק **את כל** מי שלא היה פעיל 30 יום — **רק** state, **לא** מוחק audit |

> **חשוב:** 18 / 19 / 20 / 22 עלולים למחוק או לדרוס התקדמות. הרץ רק אחרי אישור וגיבוי.

## סדר מספרים

המספור (01–22) משקף סדר שימוש יומיומי משוער: סקירות → audit → אימות סכימה → פעולות מסוכנות.

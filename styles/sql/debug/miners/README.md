# MINERS — שאילתות דיבאג (`sql/debug/miners/`)

קבצי SQL לבדיקות ותחזוקה ב-Supabase / Postgres. **כל קובץ = פעולה עיקרית אחת.**

## לפני הרצה

- בקבצים עם **`PUT-DEVICE-ID-HERE`** — החלף ב-`device_id` האמיתי.
- מומלץ **staging** לפני production.

## בטוחים לבדיקות שגרתיות (קריאה / דיווח)

קבצים **01–11** מתאימים לסריקה יומיומית:

| טווח | תוכן |
|------|------|
| **01–06** | סקירות, פעילות 24ש, top 20, תקרה יומית, “תקועים” |
| **07–10** | `miners_get_config`, economy config, multipliers, gift rewards |
| **11** | `miners_get_state` למכשיר אחד |

קבצים **18, 19, 21–25, 28, 30** — גם קריאה בלבד (דוחות, אימות עמודות, הגדרות פונקציות, vault משותף, רשימת לא פעילים).

## משנים state או הרסניים — זהירות

| קבצים | סיכון |
|--------|--------|
| **12, 13** | `miners_apply_breaks` — מוסיף mined/balance |
| **14** | `miners_claim_hourly_gift` — עלול לזרוק אם cooldown |
| **15, 16, 17** | העברות ל-vault / claim ל-wallet |
| **20** | `UPDATE` ידני ל-balance |
| **26** | מוחק **משתמש אחד** מ-miners_device_state |
| **27** | **איפוס רך** (אפסים + gift) |
| **29** | מחיקה **מאסיבית** לכל מי שלא עודכן 30 יום |

> **26** = מחיקת משתמש. **27** = איפוס רך. **29** = מחיקה המונית — להריץ רק אחרי **28** וגיבוי.

## הערות

- אין כאן טבלת audit ייעודית ל-MINERS (בניגוד ל-BASE); הדגש הוא על `miners_device_state`, קונפיג, ו-RPCים מ-`miners_server_authority.sql`.
- **30** דורש שטבלת `public.vault_balances` קיימת (סכמת vault).
